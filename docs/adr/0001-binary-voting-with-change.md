# Binary Voting with Change Capability

Votes are binary (good/bad) and users may change their vote at any time while the idea is open.

This was chosen over upvote-only (Reddit-style) because the app's purpose is to gauge idea quality from both directions — knowing an idea is disliked is as valuable as knowing it's liked. Vote changes were allowed because users should be able to update their opinion as ideas evolve or discussions unfold.

**Considered Options:**
- Upvote-only (no downvotes) — rejected because it loses negative signal
- Binary, no changes — rejected because it punishes early voters who haven't seen the full discussion
- Multi-point scale (1-5) — rejected as over-engineered for a yes/no judgment
