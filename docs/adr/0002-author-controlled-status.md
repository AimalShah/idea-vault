# Author-Controlled Status

The author of an idea exclusively controls its status, toggling between `open` and `closed`. No auto-close rules, no admin override.

This was chosen because ideas are personal — the author best knows when a discussion has run its course. Auto-close rules (time-based or vote-threshold) would impose arbitrary timelines on conversations that may still be valuable. Admin-only close would create a bottleneck and remove ownership from the author.

**Considered Options:**
- Auto-close after N days of inactivity — rejected because some ideas are long-lived
- Auto-close after vote threshold (e.g., 100 votes) — rejected because vote count doesn't indicate discussion maturity
- Admin-only close — rejected because it removes author agency
- Community vote to close (Stack Overflow style) — rejected as over-engineered for this stage
